Question-2: What is the difference between innerHTML, innerText, and textContent ?

The innerHTML property retrieves or sets the HTML content inside an element, including any nested HTML tags. On the other hand, innerText returns or modifies the visible text content of an element, excluding any HTML tags and taking CSS styles (like display: none) into account. textContent, unlike innerText, retrieves or sets all the text inside an element without considering CSS styles and is faster because it doesn't account for the rendered state of the element.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a technique where a single event listener is added to a parent element to manage events for child elements. Instead of attaching an event listener to each individual child, the parent element listens for events that bubble up from the child elements, and then determines which child was the target of the event.

Question-4: What is event bubbling in the DOM?

Event bubbling is a concept in the DOM where an event triggered on an element propagates upward through the DOM hierarchy, starting from the target element and moving towards the root. This means that if an event occurs on a nested element, it will trigger all the event listeners on its parent elements as well, unless propagation is stopped.

Question-5: How do you create, add, and remove elements using JavaScript?

To create elements using JavaScript, I can use the document.createElement() method, which generates a new HTML element. Once created, I can append the element to the DOM using appendChild() or insertBefore(), specifying the parent element where it should be added. To remove an element, I can use the removeChild() method on the parent element or remove() on the element itself to delete it from the DOM.
